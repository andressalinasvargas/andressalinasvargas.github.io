Partes de un proyecto web

Un proyecto de software web se organiza en capas con responsabilidades bien definidas que se comunican entre sí.

Capa 01
Frontend
Capa de presentación

La parte visible del sistema. Muestra información y captura la interacción del usuario en el navegador.

HTML5 CSS3 JavaScript React Vue.js
HTML — Estructura
Define elementos: títulos, formularios, botones e imágenes.
CSS — Estilos
Controla colores, fuentes, espaciado y diseño responsivo.
JavaScript — Interactividad
Validaciones, llamadas al servidor y actualizaciones dinámicas.
Diseño responsivo
Adapta la interfaz a cualquier tamaño de pantalla.
Capa 02
Backend
Capa de lógica

El servidor que procesa solicitudes, aplica las reglas del negocio y conecta con la base de datos.

PHP Node.js Python Java
Autenticación
Verifica identidad del usuario y controla acceso por roles.
Lógica de negocio
Aplica reglas, cálculos y validaciones del sistema.
API REST
Expone endpoints que el frontend consume en formato JSON.
Seguridad
Previene inyección SQL, XSS y otros ataques.
Capa 03
Base de datos
Capa de persistencia

Guarda toda la información del sistema de forma estructurada, segura y eficiente.

MySQL PostgreSQL MongoDB
BD Relacionales (SQL)
Datos en tablas con relaciones. Garantizan integridad y permiten consultas complejas.
BD No relacionales (NoSQL)
Datos en documentos o clave-valor. Ideales para alta escala.
Modelo entidad-relación
Diagrama que define tablas, columnas y relaciones del sistema.
Backups
Copias de seguridad programadas para proteger la información.
Capa 04
Infraestructura
Capa de despliegue

Todo lo necesario para que el sistema funcione en producción y sea accesible por los usuarios reales.

AWS Azure Docker
Servidor / Hosting
Máquina física o virtual donde vive el sistema.
Dominio y DNS
Nombre que identifica el sistema en internet (ej: sistema.com).
SSL / HTTPS
Cifra la comunicación para proteger contraseñas y datos.
Monitoreo
Detecta caídas y errores antes de que el usuario los note.
Capa 05
Documentación
Componente transversal

Acompaña todas las etapas del proyecto. Sin ella el mantenimiento y uso autónomo se vuelven imposibles.

README.md Diagramas UML Manual de usuario
Documentación técnica
Arquitectura, modelo de datos y decisiones de diseño.
Manual de usuario
Guía para que el usuario use el sistema sin ayuda técnica.
Manual de instalación
Pasos para desplegar el sistema en un nuevo servidor.
Comentarios en código
Explican la lógica compleja dentro del código fuente.
