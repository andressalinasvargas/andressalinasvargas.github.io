Fundamentos

¿Qué es la Elicitación de Requisitos?

La elicitación de requisitos es el proceso mediante el cual se identifican, recopilan y articulan las necesidades y expectativas de los stakeholders para construir un sistema de software exitoso.

"La elicitación no es simplemente preguntar qué quiere el usuario — es descubrir lo que realmente necesita."
— Ian Sommerville, Software Engineering

02
Exploración

Técnicas de Prospección

Buscan explorar activamente el dominio del problema para descubrir qué necesitan los stakeholders mediante comunicación directa.

Técnica Principal

Entrevistas

La técnica más utilizada. Consiste en sesiones de diálogo estructuradas o semiestructuradas con stakeholders clave para extraer conocimiento, expectativas y restricciones del sistema.

Estructurada No estructurada Semiestructurada
  • Preparar preguntas abiertas y cerradas
  • Grabar con consentimiento del entrevistado
  • Analizar respuestas inmediatamente
  • Validar resúmenes con el entrevistado
  • Identificar contradicciones entre versiones
📋

Cuestionarios y Encuestas

Recopilan información de múltiples stakeholders de forma paralela. Ideales cuando el número de personas es elevado.

🤝

Talleres JAD/RAD

Joint/Rapid Application Development. Sesiones colaborativas donde desarrolladores y usuarios trabajan conjuntamente en tiempo real.

📊

Análisis de Negocio

Estudio de procesos, flujos de trabajo y modelos organizacionales para identificar necesidades sistémicas alineadas con los objetivos del negocio.

03
Innovación

Técnicas de Creatividad

Cuando los stakeholders no saben exactamente qué quieren, las técnicas creativas permiten generar ideas y descubrir requisitos latentes e innovadores.

Técnica Estrella

Brainstorming

Sesión grupal donde se generan ideas sin crítica ni restricción. El objetivo es cantidad antes que calidad.

Grupal Individual 6-3-5 Brainwriting
  • No juzgar ideas durante la sesión
  • Fomentar ideas "salvajes" o innovadoras
  • Construir sobre ideas de otros
  • Registrar absolutamente todo
  • Seleccionar y refinar al final
🔄

SCAMPER

Técnica estructurada: Sustituir, Combinar, Adaptar, Modificar, Poner en otros usos, Eliminar, Reorganizar.

🃏

Analogías y Metáforas

Comparar el sistema con objetos o situaciones del mundo real para facilitar la comunicación entre técnicos y no técnicos.

🎲

Pensamiento de Diseño

Design Thinking. Proceso centrado en el humano: empatizar, definir, idear, prototipar y testear.

🗺️

Mapas Mentales

Representación visual jerárquica de conceptos. Permite explorar dominios de manera no lineal descubriendo conexiones ocultas.

04
Archivo

Técnicas Centradas en la Documentación

Extraen conocimiento de fuentes documentales existentes: manuales, normativas, sistemas legados, contratos y especificaciones previas.

Técnica Base

Arqueología de Sistemas

Análisis profundo de sistemas existentes para extraer requisitos implícitos. Esencial cuando se reemplaza un sistema heredado sin documentación actualizada.

Sistemas legacy Migración Reingeniería
  • Revisar código fuente existente
  • Analizar bases de datos en producción
  • Estudiar interfaces de usuario actuales
  • Examinar logs y reportes históricos
  • Documentar comportamientos implícitos
📜

Análisis de Documentos

Revisión sistemática de manuales, procedimientos, contratos y políticas institucionales para identificar restricciones no expresadas verbalmente.

♻️

Reutilización de Requisitos

Aprovechamiento de especificaciones de proyectos anteriores similares. Reduce tiempos y aprovecha el conocimiento organizacional acumulado.

⚖️

Análisis Normativo

Identificación de requisitos derivados de leyes, regulaciones, estándares industriales (ISO, IEEE) y marcos de cumplimiento obligatorio.

05
Campo

Técnicas de Observación

Permiten capturar el comportamiento real de los usuarios en su entorno natural, descubriendo requisitos que los stakeholders no pueden verbalizar.

"Lo que la gente dice que hace y lo que realmente hace suelen ser cosas muy diferentes."
— Principio fundamental de la Etnografía Aplicada
Técnica Central

Observación Directa (Field Study)

El analista se desplaza al lugar de trabajo del usuario y observa cómo realiza sus tareas en tiempo real, sin intervenir. Revela procesos informales y necesidades tácitas.

Pasiva Participativa Contextual
  • No interrumpir el flujo de trabajo
  • Registrar tiempos y frecuencias
  • Notar excepciones y casos atípicos
  • Documentar el entorno físico
  • Identificar herramientas informales usadas
🏘️

Etnografía

El analista se integra en el entorno del usuario durante un período prolongado. Descubre patrones culturales y normas informales en contexto real.

📝

Análisis de Protocolo

El usuario verbaliza en voz alta su proceso de pensamiento mientras realiza una tarea (Think Aloud), revelando decisiones y estrategias mentales.

🎬

Shadowing

El analista "sigue como una sombra" al usuario durante su jornada completa, capturando una visión holística de actividades, interrupciones y prioridades reales.
