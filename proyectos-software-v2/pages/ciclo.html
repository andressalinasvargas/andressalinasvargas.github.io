05

Ciclo de vida

01

Análisis de requerimientos

Se identifican las necesidades del cliente y los usuarios. Se definen qué debe hacer el sistema (requerimientos funcionales) y cómo debe comportarse (no funcionales).

EntrevistasCasos de usoHistorias de usuarioDiagramas de flujo
02

Diseño

Se planifica la arquitectura del sistema: estructura de la base de datos, diseño de la interfaz (mockups y wireframes) y organización del código fuente.

Diagramas UMLModelo ERWireframesPrototipo
03

Desarrollo (codificación)

Se escribe el código fuente del sistema. El frontend, el backend y la base de datos se implementan y conectan entre sí siguiendo el diseño definido.

HTML/CSS/JSPHP · Python · JavaSQLGit
04

Pruebas (testing)

Se verifica que el sistema funcione correctamente. Se realizan pruebas unitarias, de integración y con usuarios reales para detectar y corregir errores.

Pruebas funcionalesPruebas de usabilidadPruebas de cargaReporte de bugs
05

Despliegue

El sistema se publica en un servidor real para que los usuarios finales puedan acceder. Se configura el dominio, la seguridad y las copias de seguridad.

Hosting / ServidorDominioHTTPSBackup
06

Mantenimiento

Después de la entrega, se corrigen errores detectados en producción, se mejoran funciones y se adapta el sistema a nuevas necesidades del usuario.

ActualizacionesSoporte técnicoMejoras continuasMonitoreo
