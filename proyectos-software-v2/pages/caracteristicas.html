Propiedades del software

Un proyecto de software tiene características únicas que lo distinguen de proyectos de ingeniería física, agrupadas en técnicas y de gestión.

Técnicas
Técnica

Intangibilidad

El software no tiene forma física. Existe como código, datos y lógica ejecutable almacenada digitalmente.

Técnica

Complejidad

Involucra múltiples capas: interfaz visual, lógica de negocio y base de datos que interactúan entre sí.

Técnica

Escalabilidad

Capacidad de crecer en usuarios, datos o funciones sin perder rendimiento. Puede ser vertical u horizontal.

Técnica

Mantenibilidad

Facilidad con que el código puede corregirse y mejorarse. Un código limpio reduce los costos futuros.

Técnica

Portabilidad

Capacidad de funcionar en diferentes sistemas operativos o dispositivos sin modificaciones importantes.

Técnica

Confiabilidad

Probabilidad de que el sistema funcione correctamente durante un período determinado de operación.

De gestión
Gestión

Temporalidad

Todo proyecto tiene inicio y fin definidos. Al entregar el producto pasa a fase de mantenimiento.

Gestión

Unicidad

Ningún proyecto es idéntico a otro. Requerimientos, contexto y equipo siempre son diferentes.

Gestión

Iteratividad

Se construye en ciclos. Cada iteración agrega funcionalidades y permite incorporar retroalimentación.

Gestión

Dependencia humana

El éxito depende del equipo. Habilidades técnicas, comunicación y motivación son factores clave.

Gestión

Evolución continua

El software debe adaptarse constantemente a nuevas necesidades del negocio y la tecnología.

Gestión

Riesgo inherente

Todo proyecto conlleva incertidumbre: requerimientos ambiguos, tecnología nueva o estimaciones incorrectas.
